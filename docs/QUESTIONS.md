## 領域①　経営・宣言（5問・重み20%）

> 経営層がキャリアオーナーシップを事業戦略と接続して発信しているかを測る

**Q1.**
経営層（CEO・CHROなど）が、キャリア自律の重要性を公式な場で繰り返し発信している

**Q2.**
キャリアオーナーシップに関する方針が、中期経営計画や人材戦略の文書に明記されている

**Q3.**
「なぜキャリア自律が必要か」について、経営と人事の間で共通の言語・定義が存在している

**Q4.**
経営層が、社員のキャリア自律を「経営投資」として捉えて意思決定していると感じる

**Q5.**
キャリアオーナーシップの推進が、事業成果や人的資本KPIと紐づけて語られている

## 領域②　管理職支援（5問・重み20%）

> 管理職がメンバーのキャリアオーナーシップを支援する行動をとっているかを測る

**Q6.**
管理職が、業務進捗以外にメンバーのキャリアについて対話する機会を定期的に設けている

**Q7.**
管理職が、メンバーの「やりたいこと（Will）」を把握したうえで育成・配置に関わっている

**Q8.**
管理職自身が自分のキャリアを主体的に考えており、メンバーへの支援姿勢に表れていると感じる

**Q9.**
管理職が、メンバーの挑戦や異動希望を部門の損失として抵抗するのではなく、後押しする姿勢が組織に根付いている

**Q10.**
管理職に対して、キャリア支援のスキルや対話方法に関するトレーニングが提供されている

## スコアリング計算式（実装用）

### 正規化計算式

```javascript
// 領域スコアの計算（0〜100点）
function calcAreaScore(answers) {
  const n = answers.length;           // 設問数
  const sum = answers.reduce((a, b) => a + b, 0); // 回答合計
  return (sum - n) / (n * 5) * 100;
}

// 総合スコアの計算
function calcTotalScore(areaScores) {
  const weights = {
    leadership:  0.20,  // 経営・宣言
    manager:     0.20,  // 管理職支援
    culture:     0.20,  // 組織風土
    system:      0.15,  // 制度設計
    individual:  0.15,  // 個人行動
    data:        0.10   // データ把握
  };

  return Object.keys(weights).reduce((total, key) => {
    return total + areaScores[key] * weights[key];
  }, 0);
}

// 段階判定
function getStage(score) {
  if (score < 20) return { level: 1, name: '黎明期' };
  if (score < 40) return { level: 2, name: '認知期' };
  if (score < 60) return { level: 3, name: '理解期' };
  if (score < 80) return { level: 4, name: '共感期' };
  return { level: 5, name: '実践期' };
}
```

### 計算例（全問3回答）

| 領域 | 設問数 | 合計 | 計算式 | 領域スコア |
|------|--------|------|--------|-----------|
| 経営・宣言 | 5 | 15 | (15-5)/(5×5)×100 | 40.0点 |
| 管理職支援 | 5 | 15 | (15-5)/(5×5)×100 | 40.0点 |
| 組織風土 | 5 | 15 | (15-5)/(5×5)×100 | 40.0点 |
| 制度設計 | 4 | 12 | (12-4)/(4×5)×100 | 40.0点 |
| 個人行動 | 4 | 12 | (12-4)/(4×5)×100 | 40.0点 |
| データ把握 | 3 | 9 | (9-3)/(3×5)×100 | 40.0点 |
| **総合** | | | 40.0×1.00 | **40.0点** |
| **段階** | | | | **理解期** |

## 結果コメント定型文の設計方針

### 強み・課題コメントの自動生成ロジック

- 領域別スコアを降順に並べ、**上位2領域を「強み」**、**下位2領域を「課題」**として表示
- 同スコアの場合は優先度（高→中→低）の順で決定

### NODIAからの提言コメント（別途作成）

段階と弱点領域の組み合わせごとに定型文を作成する。

| 段階 | 弱点領域 | 提言の方向性 |
|------|---------|------------|
| 黎明期 | 経営・宣言 | 経営層への働きかけ方・根拠づくり |
| 黎明期 | 管理職支援 | 管理職育成の入口施策 |
| 認知期 | 組織風土 | 心理的安全性の醸成アプローチ |
| 認知期 | 制度設計 | 制度設計の優先順位と設計手法 |
| 理解期 | 個人行動 | 社員の行動変容を促す施策設計 |
| 理解期 | 管理職支援 | 管理職のC/O支援行動診断の活用 |
| 共感期 | データ把握 | 人的資本KPIとC/Oの接続 |
| 共感期 | 経営・宣言 | 経営戦略とC/Oの接続・数値化 |
| 実践期 | 全領域 | さらなる高度化・他社比較の活用 |

※ 上記は方向性の骨格。具体的な文章はNODIAコンサルタントが別途作成する。
